Resolved this session: the "Call Control Service (location TBD)" node below is now call-engine — a Node.js service, its own Kubernetes pod, deployed from the Velents-Technologies-UG/agent-hub repo. A real end-to-end call test proved the whole live path (grey solid lines) works correctly, including this node. One open wrinkle: that repo's main branch is itself a frontend app, with no visible trace of this backend at its root — full detail in the companion "Worker 2" architecture doc. The velents_integrations node's "proposed" hop is now a decided design (synced-copy model), also detailed there.
